# 장애복구 절차 & 배포 (jbond.iamchart.co.kr)

## 장애복구 (원천 서비스 장애)
1. API `createPrimaryAdapter` 가 `live` 실패 시 예외 → 상위에서 캐치, 프런트는 **마지막 정상 데이터**(localStorage `jbond.lastgood.v1`) 표시 + `StaleBanner`(갱신시각) 노출.
2. 원천 오류는 `data_ingestion_logs` 에 status=ERROR 로 기록.
3. 복구 순서:
   - 원천 상태 확인(SEIBro/KOFIA/KRX)
   - 어댑터 헬스체크 `GET /health`
   - 재수집 배치 재실행
   - 결측 구간은 보간/대체값으로 표시(임의 0 대체 금지)
4. Service Worker(NetworkFirst) 가 API 캐시를 7일 보관 → 오프라인 마지막 조회 지원.

## 롤백
- 프런트: 직전 dist 아티팩트로 교체(정적 호스팅)
- API: 직전 이미지/커밋으로 롤백, DB 마이그레이션은 순방향만(파괴적 변경 금지)

## 배포 절차
1. 빌드
   ```
   npm install
   npm run build          # packages → web(dist)
   ```
2. API
   ```
   # .env 주입 후
   npm run start --workspace apps/api    # tsx src/server.ts
   ```
3. 프런트(dist) 를 정적 호스팅/CDN 업로드, `jbond.iamchart.co.kr` 연결
4. HTTPS 인증서, Rate Limit, 입력검증 확인
5. PWA 설치 테스트(홈 화면 추가), 오프라인 마지막 조회 확인
6. 실기기(360px) 스모크 테스트

## 감사성
- 계산 입력·엔진 버전·출처를 `simulation_results.engine_version` 및 `source_metadata` 에 기록.
