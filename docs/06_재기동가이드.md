# __AI_JBOND__ 프로젝트 재기동 가이드

프로젝트 루트: `G:\__AI_JBond__`
모노레포(npm workspaces) — `apps/web`, `apps/api`, `packages/{shared-types,bond-engine,ui}`

---

## 0. 사전 요구
- **Node.js ≥ 20** (확인: `node -v`) · npm ≥ 9
- Windows에서 **안티바이러스 실시간 검사 예외**로 `G:\__AI_JBond__` 등록 권장
  (미등록 시 `npm install` 이 `EBUSY: resource busy or locked` 로 실패할 수 있음)

## 1. 클린 설치 (권장)
```bash
cd /g/__AI_JBond__          # PowerShell: cd G:\__AI_JBond__
rm -rf node_modules packages/*/node_modules apps/*/node_modules package-lock.json
npm install                 # 캐시가 있으면 --prefer-offline 로 가속
```
> 네트워크가 느리면: `npm install --fetch-retries=5 --fetch-retry-maxtimeout=60000 --prefer-offline`
> 캐시가 이미 채워진 경우 완전 오프라인: `npm install --offline`

## 2. 개발 서버 기동
```bash
# 웹앱(모바일 PWA) — 백엔드 없이 내장 Mock 으로 동작
npm run dev            # → http://localhost:5173

# (선택) API 서버 — Source Adapter
npm run dev:api        # → http://localhost:4000  (.env 필요)
```
- 웹앱만으로 5개 화면(발행/곡선/유통/투자/시가표) 전부 동작 (임베디드 Mock).
- API 연동하려면 `apps/web` 에서 `VITE_API_BASE_URL=http://localhost:4000` 설정.

## 3. 검증
```bash
npm run test           # bond-engine + web 단위 테스트
npm run build          # 타입체크 + Vite PWA 프로덕션 빌드
npm run test:e2e       # Playwright E2E (360/390/430)  ※ 최초 1회: npx playwright install chromium
```
계산 엔진만 빠르게 검증(외부 의존 없음):
```bash
cd packages/bond-engine && npx vitest run     # 26 테스트
```

## 4. 프로덕션 미리보기
```bash
npm run build
npm run preview --workspace apps/web   # → http://localhost:4173
```

## 5. 설치 없이 UI 확인
`docs/ui-preview.html` 을 브라우저로 열기 — 실제 bond-engine 계산·Mock 데이터를 포팅한 5개 화면 인터랙티브 프리뷰.

## 6. 환경변수 (.env)
`\.env.example` 복사 → `.env` 작성 (비밀키 코드 하드코딩 금지). 주요 항목:
| 변수 | 용도 |
|---|---|
| `DATA_SOURCE_MODE` | `mock`(기본) / `live` |
| `SEIBRO_BASE_URL` / `SEIBRO_API_KEY` | 기본 원천 |
| `KOFIA_*`, `KRX_*` | 최종호가/민평, 실거래 보완 |
| `DATABASE_URL` / `PG*` | PostgreSQL |
| `VITE_API_BASE_URL` | 프런트→API (비우면 임베디드 Mock) |

## 7. DB (실원천 연동 단계)
```bash
psql "$DATABASE_URL" -f db/migrations/001_init.sql
```

## 자주 겪는 문제
| 증상 | 원인 | 해결 |
|---|---|---|
| `EBUSY ... rmdir node_modules\.<pkg>-<hash>` | AV가 npm 임시 staging 잠금 | AV 예외 등록 → `rm -rf node_modules` → 재설치 |
| `npm install` 이 매우 느림 | 느린 네트워크 | `--prefer-offline` / `--offline` (캐시 재사용) |
| `vite: command not found` | 설치 미완료(bin 링크 안 됨) | 클린 재설치 (1번) |
| 웹앱에 데이터 안 뜸 | — | 기본이 Mock 이라 백엔드 불필요. `VITE_API_BASE_URL` 비우기 |
