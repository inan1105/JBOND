# 데이터 출처 문서

## 원천
| 출처 | 역할 | valueType | 상태 |
|---|---|---|---|
| SEIBro | 기본 원천. 시장현황·종목검색·만기수익률·유통추정·평가수익률 | MARK_TO_MARKET / ISSUE_YIELD | WebSquare 규격 확인 필요(TODO) |
| KOFIA(금투협) | 최종호가수익률·민평수익률 | FINAL_QUOTE | API 규격 확인 필요(TODO) |
| KRX/금융위 | 실거래 시가·고저종가·거래량·수익률 | ACTUAL_TRADE | 공공 API 연동(TODO) |
| Mock | 개발/데모 기본값 | 전체 | **구현 완료** |

## 수집 아키텍처
```
모바일 → James Bond API → Source Adapter → SEIBro / KOFIA / KRX
```
프런트는 원천을 직접 호출하지 않는다(CORS·세션·WebSquare 규격 변경·호출량 제한 통제).
`SeibroAdapter / KofiaAdapter / KrxAdapter / MockBondDataAdapter` 로 분리하여 원천 변경이 UI에 영향을 주지 않게 한다.

## 어댑터 필수 메타
각 관측치는 `source, sourceUrl, sourceTimestamp, collectedAt, valueType, qualityStatus, rawReference` 저장.
결측값은 **0으로 대체하지 않고 null 유지**.

## 실원천 연동 절차 (1단계: 데이터 탐색·검증)
1. SEIBro 화면 브라우저 네트워크 패널로 실제 조회 요청·응답 필드 확인
2. 사용 조건·호출 제한 확인
3. 발행정보 필드 매핑표 작성
4. 개별 평가수익률 확보 가능성 확인
5. 과거 수익률곡선 확보 기간 확인
6. 결측·휴일 처리원칙 확정
7. `SeibroAdapter` 의 TODO 를 실제 매핑으로 대체, `DATA_SOURCE_MODE=live` 전환

## 대체값 우선순위 (유통 수익률)
1. 시가평가수익률 → 2. 실제 거래수익률 → 3. 발행수익률 → 4. 동종 Curve → 5. 사용자 입력
3~5는 화면에 **대체값**으로 명시한다.

## 최종호가 주의
최종호가수익률은 대표물 중심이므로 전체 등급×만기 표는 민평수익률과 결합. 행별 출처 배지(최종호가·민평·실거래·보간·사용자) 부착.
