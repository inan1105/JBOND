# 제임스 본드(James Bond) — 프로젝트 분석 및 실행계획 (Prompt01 산출물)

## 1. 목표 요약
모바일 전용 PWA로 하나의 `bondId`를 축으로 **발행 → 유통 → 곡선 → 시가표 → 투자시뮬레이션** 5개 콘텐츠를 연결한다.
핵심은 "5개 메뉴를 각각 만드는 것"이 아니라 **채권 마스터 → 시장수익률 → 가격 계산 → 투자성과**로 이어지는 단일 데이터 파이프라인 구축이다(PRD/Prompt99).

## 2. 아키텍처 결정

| 계층 | 선택 | 근거 |
|---|---|---|
| 프런트 | React + TypeScript + Vite | PRD 권장 |
| 스타일 | Tailwind CSS | 모바일 390px 기준, 유틸리티 |
| 차트 | Apache ECharts | PRD 권장, 곡선/스프레드/궤적 |
| 표 | 직접 구현 Sticky Grid | 시가표 헤더·첫열 고정 요구 |
| 상태 | Zustand (BondContext) | 화면 간 전달, 경량 |
| 백엔드 | Node.js + Fastify (TS) | PRD 권장, Source Adapter 격리 |
| DB | PostgreSQL (스키마/마이그레이션) | PRD 권장 (MVP는 Mock 우선) |
| 순수계산 | packages/bond-engine (무상태 TS) | 회귀테스트 대상 |
| 테스트 | Vitest + Playwright | 단위/E2E |
| PWA | vite-plugin-pwa (SW + manifest) | 홈설치, 오프라인 마지막 조회 |

**데이터 흐름:** 모바일 → James Bond API → Source Adapter(Seibro/Kofia/Krx/Mock) → 원천.
프런트는 SEIBro를 직접 호출하지 않는다(CORS·세션·WebSquare 규격 변경 통제).

## 3. 모노레포 구조
```
apps/
  web/            React PWA (5 화면 + 공통 셸)
  api/            Fastify API + Source Adapter
packages/
  shared-types/   BondMaster, BondTerms, ... BondContext
  bond-engine/    순수 단가 계산 + 테스트
  ui/             공통 모바일 컴포넌트 (카드/배지/시트/스켈레톤)
docs/             분석·계산식·출처·운영 문서
```

## 4. 핵심 설계 원칙 (PRD §1.3 반영)
- 결측값은 **0으로 대체 금지 → null 유지**.
- 관측값 / 계산·보간·추정값을 **valueType·qualityStatus 배지**로 구분.
- 모든 수익률·단가에 **기준일·출처** 표시.
- 화면 간 재입력 없이 **BondContext**로 전달.
- 모바일: 핵심 숫자 3개 우선, 상세는 아코디언.

## 6. 리스크 및 대응
- **SEIBro WebSquare 규격 미확인** → 실제 요청은 TODO+Mock으로 격리, 어댑터 인터페이스로 흡수.
- **과거 곡선 데이터 확보** → Mock 시계열 생성기로 대체, 실데이터 연동은 어댑터 교체.
- **모바일 표 z-index 충돌** → sticky 헤더/첫열 교차셀 최상위 레이어 규칙 확정.
- **세법 자동판정 금지** → 세율은 사용자 설정값(`jbond.tax.v1`).
